在Hexo的next主题下使用Valine评论系统

发表于 2019-04-17 | 更新于 2019-04-18 | 评论数： | 阅读次数：

虽然现在还没有读者，不过既然开了Hexo博客，就得尝试下开启评论功能。简单搜索了一下，确定了Valine。

Valine

首先Valine是一套国产的评论系统，所以不存在翻墙问题。它是基于LeanCloud（我也没去了解）的。主要特点是支持Markdown语法，无后端实现（也就是不需要登录），并且支持文章阅读量统计，另外比较简洁美观，这几点是我选择它的主要原因。另外根据Valine官网介绍，它还具备快速、安全、轻量并且支持Emoji的特点。

本博客使用的是next主题，内置支持了Valine评论系统，所以使用起来很方便。

获取APP ID 和 APP Key

毕竟是使用别人的服务，所以这一步需要注册登录LeanCloud。具体操作看官方文档。

注册并创建应用后就可以拿到APP ID和APP Key了，然后就可以在next主题内配置使用Valine评论系统了。

在next主题的_config.yml文件中配置

在编辑器打开next主题目录下的_config.yml文件，直接Ctrl+F搜索Valine，就可以找到Valine的配置项并进行配置了（ps：next还支持其他评论系统，配置项就在附近）。

来看一下配置项：

最重要的就是要配置enable要设为true，然后要正确配置appid和appkey，用hexo重新发布之后评论系统就可以用了。

补充说明，visitor表示是否开启记录这篇文章的被浏览次数的功能，comment_count则是开始评论计数功能，这二者会在文章标题下的元信息部分添加评论数和阅读次数两个内容。

具体效果可以看这篇文章。

hexo评论的开启和关闭

首先，整站的评论系统的开启和关闭是在主题的_config.yml文件内配置。

单篇博客文章的评论功能默认是开启的，要关闭单篇博客的评论功能，则是在博客的Front-matter部分添加comments: false字段。

Valine的主要缺点

我暂时觉得Valine的主要缺点在于无法登录。

不用登录就能评论既是优点又是缺点。优点就不用多说了，不强求用户登录，可匿名，所以想评就评，用户没有成本。

缺点是除了可以匿名，用户完全还可以输入假的个人信息，如果你想和对方深入沟通，不一定真能找到对方；另外头像配置麻烦；没有登录验证自然也就存在安全隐患（不过Valine自己宣称是安全的，我没研究），至少更容易被灌水、灌垃圾评论等。

ps：我原本以为这种方案不会记录用户信息，后来发现是有记录用户输入的个人信息的，通过localStorage存储下来了。

替代方案

我目前没做仔细研究，所以直接从next主题的_config.yml中提取出它支持的评论系统吧。

  • Disqus
  • DisqusJS
  • Changyan
  • Valine
  • Gitment
  • Gittalk

其中Gitment和Gittalk都是效果看起来都还不错，Gittalk好看点。关键是它们是基于Github的，用户用的是github账户登录，所以如果要了解评论者信息、和对方深入沟通，这就很方便了。对于技术博客而言，这两者都是不错的选择。

It is used to create components, handle state and props, utilize event listeners and certain life cycle methods to update data as it changes.

TopoJSON介绍

TopoJSON是使用拓扑结构编码来实现的一个GeoJSON的扩展。官方原文是这样的：”TopoJSON is an extension of GeoJSON that encodes topology.”具体的介绍可以看官方文档。

总之TopoJSON相对于GeoJSON有很大优势：消除了大量冗余（比如边界地理数据重复等），使得描述同一地理的TopoJSON文件比GeoJSON文件体积小了很多；就绘制而言，使用TopoJSON可以做到仅绘制外围轮廓、共享边境不重复绘制、查找相邻地区等。
